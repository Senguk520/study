# C++17 万能基础竞赛模板

## 适用场景

这份模板适合 ACM-ICPC / CCPC / 蓝桥杯等算法竞赛的基础题、模拟题、数学题、搜索题、图论题等常见场景。

特点：

- 使用 C++17 标准。
- 使用 `#include <bits/stdc++.h>` 快速引入常用库。
- 使用快速 I/O，避免大量输入输出导致超时。
- 使用 `solve()` 函数处理每一组独立测试数据。
- 默认支持多组测试数据。

## 基础模板代码

```cpp
#include <bits/stdc++.h>          // 万能头文件，包含竞赛中常用的大部分标准库；编译期略慢，但竞赛写代码更快
using namespace std;              // 使用标准命名空间，避免每次写 std::，减少代码量

using ll = long long;             // 给 long long 起别名 ll；用于防止 int 溢出，常见于求和、乘法、计数

/*
    solve()：处理一组独立测试数据

    时间复杂度：由题目逻辑决定，假设单组数据复杂度为 O(f(n))

    多组数据常见坑点：
    1. solve() 内部的局部变量每次调用都会重新创建，一般不需要手动清空
    2. 如果使用全局数组、全局 vector、map、set、vis 标记数组，必须在每组数据前后清空
    3. 注意数据范围，求和、乘法、答案变量优先考虑 long long
*/
void solve() {                    // 每调用一次 solve()，处理一组测试数据；单次复杂度取决于题目
    // 在这里写每组测试数据的核心代码

    // 示例：读入 n 和长度为 n 的数组
    // int n;                     // 定义整数 n；O(1)
    // cin >> n;                  // 读入 n；O(1)
    // vector<int> a(n);          // 创建长度为 n 的数组；O(n)
    // for (int i = 0; i < n; i++) { // 循环 n 次；O(n)
    //     cin >> a[i];           // 读入第 i 个数；总复杂度 O(n)
    // }

    // 示例：输出答案
    // cout << ans << '\n';       // 使用 '\n' 换行；O(1)，比 endl 更快
}

int main() {                      // 程序入口；O(1)
    ios::sync_with_stdio(false);  // 关闭 C++ 流和 C 流同步，加速 cin/cout；O(1)
    cin.tie(0);                   // 解除 cin 与 cout 绑定，避免输入前自动刷新输出缓冲区；O(1)

    int T = 1;                    // 测试数据组数，默认 1 组；O(1)
    cin >> T;                     // 如果题目给出多组数据，读入 T；O(1)

    while (T--) {                 // 循环 T 次，总复杂度为 O(T * f(n))
        solve();                  // 处理当前这一组测试数据；单次复杂度 O(f(n))
    }

    return 0;                     // 程序正常结束；O(1)
}
```

## 为什么竞赛中用 `\n` 替代 `endl`

在 C++ 中：

```cpp
cout << ans << '\n';
```

表示只输出一个换行符。

而：

```cpp
cout << ans << endl;
```

不仅会输出换行符，还会强制刷新输出缓冲区。

刷新缓冲区是比较慢的操作。如果程序需要输出很多行，比如 \(10^5\) 行、\(10^6\) 行，频繁使用 `endl` 可能显著拖慢程序，甚至导致超时。

所以竞赛中普通换行推荐使用：

```cpp
cout << ans << '\n';
```

只有在交互题中，必须立刻让评测机看到输出时，才可能需要使用 `endl` 或 `cout.flush()`。

## 常见竞赛坑点提醒

### 1. `int` 溢出

`int` 范围大约是 \(-2 \times 10^9\) 到 \(2 \times 10^9\)。

如果题目中出现：

- \(n \le 10^5\)，\(a_i \le 10^9\)，要求数组总和；
- 两个 \(10^9\) 级别的数相乘；
- 答案可能达到 \(10^{18}\)；

就应该使用 `long long`。

```cpp
long long sum = 0;
```

不要写：

```cpp
int sum = 0;
```

### 2. 多组数据没有清空状态

如果变量写在 `solve()` 内部，通常每组会自动重新创建。

但如果使用全局变量，例如：

```cpp
vector<int> g[100005];
bool vis[100005];
```

多组测试数据时，必须清空：

```cpp
for (int i = 1; i <= n; i++) g[i].clear();
fill(vis + 1, vis + n + 1, false);
```

否则上一组数据会污染下一组数据。

### 3. 输入输出超时

大量输入输出时必须使用快速 I/O：

```cpp
ios::sync_with_stdio(false);
cin.tie(0);
```

并尽量使用 `\n`，不要频繁使用 `endl`。

### 4. 题目没有多组数据时怎么办

如果题目没有给出 `T`，把下面这一行注释掉：

```cpp
cin >> T;
```

此时程序默认只执行一组：

```cpp
int T = 1;
```
