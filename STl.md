## 1. `std::vector` 常用操作

### 1.1 作用说明

`vector` 可以理解为“**长度可变的数组**”。

- **优点**：支持下标访问，尾部插入快，排序方便。
- **常见用途**：存数列、存图的邻接表、存答案、离散化后去重。
- **复杂度印象**：
  - 尾部插入 `push_back()`：**均摊 \(O(1)\)**
  - 下标访问 `a[i]`：**\(O(1)\)**
  - 排序 `sort()`：**\(O(n \log n)\)**

### 1.2 示例代码

```cpp
#include <bits/stdc++.h> // 竞赛中常用万能头文件，包含大部分 STL
using namespace std;

int main() {
    // ---------- 1) 初始化 ----------
    vector<int> a;                  // 空 vector，当前没有元素
    vector<int> b = {5, 2, 2, 9, 1}; // 直接用列表初始化，竞赛里非常常见
    vector<int> c(5);               // 创建 5 个元素，默认值都是 0
    vector<int> d(5, 7);            // 创建 5 个元素，每个值都是 7

    // 复杂度：
    // vector<int> a;       是 O(1)
    // vector<int> b = {...} 需要把每个元素放进去，是 O(n)
    // vector<int> c(5) / d(5, 7) 也需要创建 n 个元素，是 O(n)

    // ---------- 2) 尾部插入 ----------
    a.push_back(10); // 在末尾加入 10
    a.push_back(20); // 在末尾加入 20
    a.push_back(30); // 在末尾加入 30

    // push_back 的复杂度是“均摊 O(1)”：
    // 大多数时候都很快，但扩容时会搬运旧元素。
    // 不过从总摊还分析看，连续插入 n 次总复杂度是 O(n)。

    // ---------- 3) 范围 for 循环遍历 ----------
    // 这里演示范围 for，这是竞赛里最常用的遍历写法之一。
    for (int x : a) {
        cout << x << ' '; // 依次输出 a 中每个元素
    }
    cout << '\n';

    // 复杂度：遍历整个 vector，需要看每个元素一次，所以是 O(n)

    // ---------- 4) 从小到大排序 ----------
    sort(b.begin(), b.end());
    // sort 默认是升序（从小到大）
    // 复杂度：O(n log n)

    for (int x : b) {
        cout << x << ' ';
    }
    cout << '\n';

    // ---------- 5) 从大到小排序 ----------
    sort(b.begin(), b.end(), greater<int>());
    // 第三个参数 greater<int>() 表示比较规则改为“大数排前面”
    // 复杂度仍然是 O(n log n)

    for (int x : b) {
        cout << x << ' ';
    }
    cout << '\n';

    // ---------- 6) 去重（unique） ----------
    vector<int> e = {4, 2, 2, 4, 1, 1, 3, 3, 3};

    sort(e.begin(), e.end());
    // 先排序，复杂度 O(n log n)
    // 为什么必须先排序？
    // 因为 unique 只能“去掉相邻的重复元素”，
    // 如果相同元素没挨在一起，它就去不干净。

    e.erase(unique(e.begin(), e.end()), e.end());
    // unique 会把“去重后应该保留的部分”挪到前面，
    // 并返回“新逻辑结尾”的迭代器。
    // 但它不会真正删掉尾巴上的多余元素。
    // 所以还必须再接一个 erase 才算真正删掉。
    // unique 的复杂度是 O(n)
    // erase 删除尾部这一段的复杂度也是 O(n) 量级
    // 整个“排序 + 去重”流程总复杂度是 O(n log n)

    for (int x : e) {
        cout << x << ' ';
    }
    cout << '\n';

    // ---------- 7) 常见坑点提醒 ----------
    // 坑 1：unique 之前一般先 sort，否则不能完全去重。
    // 坑 2：unique 不会直接删元素，必须配合 erase 使用。
    // 坑 3：多组数据时，如果 vector 是全局变量，记得 clear()。
    // 坑 4：如果数据范围很大，例如元素值达到 1e18，类型要开 long long。
    // 坑 5：如果你提前知道元素个数很多，可以用 reserve() 减少扩容次数。

    return 0;
}
```

## 2. `std::map` 和 `std::unordered_map` 的区别

### 2.1 一句话理解

- `map`：**有序**键值对容器，底层通常是**红黑树**。
- `unordered_map`：**无序**键值对容器，底层通常是**哈希表**。

### 2.2 对比表

| 容器 | 是否有序 | 插入/查找复杂度 | 适合场景 | 竞赛坑点 |
| --- | --- | --- | --- | --- |
| `map` | 按键从小到大有序 | **\(O(\log n)\)** | 需要有序遍历、前驱后继、离散键统计 | 常数略大，但稳定 |
| `unordered_map` | 无序 | **平均 \(O(1)\)**，**最坏 \(O(n)\)** | 只关心查找/计数快，不关心顺序 | 哈希可能退化，输出顺序不能依赖 |

### 2.3 什么时候用谁？

- **如果你需要按键从小到大输出**，用 `map`。
- **如果你只想做计数、判重、映射，且不关心顺序**，一般优先 `unordered_map`。
- **如果题目卡哈希、数据很毒瘤，担心退化**，用 `map` 更稳。

### 2.4 最关键区别

- `map` 遍历出来的键值对是**有序的**。
- `unordered_map` 遍历出来的顺序是**不固定的**，不要把它当成功能依赖。

## 3. 统计一段文本中每个单词出现次数

下面给一个竞赛风格的例子：统计一句英文里每个单词出现了几次。

### 3.1 用 `map` 统计（输出有序）

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    string text = "apple banana apple orange banana apple";
    // 这里假设：
    // 1. 单词之间只用空格分隔
    // 2. 不处理标点
    // 3. 不处理大小写统一
    // 这样更贴近竞赛中的“最小可用写法”

    map<string, int> cnt;
    // map 的键是 string，值是 int
    // 作用：cnt[word] 表示单词 word 出现了多少次

    stringstream ss(text);
    // stringstream 可以把一整行字符串当成输入流来拆词

    string word;
    while (ss >> word) {
        cnt[word]++;
        // 如果 word 第一次出现，cnt[word] 会先默认变成 0，再加 1
        // map 的单次访问/插入复杂度是 O(log n)
    }

    // 使用结构化绑定 + 范围 for 遍历 map
    // 这是 C++17 的重点语法之一，竞赛里很好用。
    for (auto [key, value] : cnt) {
        cout << key << " : " << value << '\n';
    }
    // 遍历全部键值对需要 O(n)
    // 其中 key 是单词，value 是出现次数

    return 0;
}
```

### 3.2 用 `unordered_map` 统计（平均更快）

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    string text = "apple banana apple orange banana apple";
    unordered_map<string, int> cnt;
    stringstream ss(text);
    string word;

    while (ss >> word) {
        cnt[word]++;
        // unordered_map 单次插入/查找平均 O(1)
        // 但最坏情况下可能退化到 O(n)
    }

    for (auto [key, value] : cnt) {
        cout << key << " : " << value << '\n';
    }
    // 遍历所有元素是 O(n)
    // 注意：这里输出顺序不保证固定

    return 0;
}
```

### 3.3 竞赛中要注意什么？

- **如果题目要求按字典序输出单词**：直接用 `map`。
- **如果题目只要求统计次数，不关心输出顺序**：`unordered_map` 平均更快。
- **如果文本很大**：
  - 仍然优先考虑总复杂度；
  - 需要注意字符串本身拷贝也有代价；
  - 如果题目涉及大小写、标点清洗，要额外写预处理。
- **如果是多组数据**：记得 `cnt.clear()` 或重新定义容器，避免上一组数据残留。

## 4. `std::set` 的插入与查找

### 4.1 作用说明

`set` 可以理解为“**自动有序且自动去重的集合**”。

- 插入相同元素多次，集合里只保留一个。
- 元素默认从小到大排列。
- 很适合做：判重、去重、快速查某个数是否出现过。

### 4.2 示例代码

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    set<int> s;

    // ---------- 1) 插入 ----------
    s.insert(5);
    s.insert(2);
    s.insert(8);
    s.insert(5); // 重复插入 5，不会报错，但集合里仍然只会保留一个 5

    // set 的 insert 复杂度是 O(log n)
    // 因为底层通常是平衡二叉搜索树

    // ---------- 2) 范围 for 遍历 ----------
    for (int x : s) {
        cout << x << ' ';
    }
    cout << '\n';
    // 遍历全部元素复杂度是 O(n)
    // 输出结果会自动按从小到大排列

    // ---------- 3) 查找 ----------
    if (s.find(2) != s.end()) {
        cout << "2 在集合里\n";
    } else {
        cout << "2 不在集合里\n";
    }
    // find 的复杂度是 O(log n)

    // ---------- 4) 判存在的另一种写法 ----------
    if (s.count(8)) {
        cout << "8 在集合里\n";
    }
    // 对 set 来说，count(x) 的结果只可能是 0 或 1
    // 复杂度同样是 O(log n)

    return 0;
}
```

### 4.3 `set` 的竞赛用法总结

- **判重**：读一个数时，先看 `set` 里有没有。
- **自动去重**：把所有数丢进去，最后天然不重复。
- **有序输出**：遍历时自动升序。

### 4.4 常见坑点

- `set` **不能像数组一样随机下标访问**，没有 `s[0]` 这种写法。
- 如果你只想要“判重 + 平均更快”，也可以考虑 `unordered_set`，它平均 **\(O(1)\)**，但最坏可能退化成 **\(O(n)\)**。
- 如果题目需要“第 k 小”“排名”，普通 `set` 不直接支持，要换别的做法。

## 5. 一眼记住的复杂度总结

### 5.1 `vector`

- `push_back()`：**均摊 \(O(1)\)**
- `a[i]` 下标访问：**\(O(1)\)**
- `sort()`：**\(O(n \log n)\)**
- `unique()`：**\(O(n)\)**
- 遍历：**\(O(n)\)**

### 5.2 `map`

- 插入：**\(O(\log n)\)**
- 查找：**\(O(\log n)\)**
- 修改：**\(O(\log n)\)**
- 遍历：**\(O(n)\)**

### 5.3 `unordered_map`

- 插入：**平均 \(O(1)\)**，最坏 **\(O(n)\)**
- 查找：**平均 \(O(1)\)**，最坏 **\(O(n)\)**
- 修改：**平均 \(O(1)\)**，最坏 **\(O(n)\)**
- 遍历：**\(O(n)\)**

### 5.4 `set`

- 插入：**\(O(\log n)\)**
- 查找：**\(O(\log n)\)**
- 删除：**\(O(\log n)\)**
- 遍历：**\(O(n)\)**

## 6. 竞赛常见坑点总复习

- **`unique` 不是删除函数**：它只是把“不重复部分”挪到前面，真正删除还要接 `erase`。
- **去重前通常先排序**：否则重复元素不相邻，`unique` 去不干净。
- **`unordered_map` / `unordered_set` 不保证有序**：不要依赖它们的遍历顺序。
- **哈希表会退化**：极端情况下 `unordered_map` 可能从平均 **\(O(1)\)** 退化到 **\(O(n)\)**。
- **多组数据要清空容器**：`vector`、`map`、`set`、`unordered_map` 用完要 `clear()`，或者在每组数据内重新定义。
- **注意数据范围**：如果数值可能达到 `1e18`，请使用 `long long`，不要乱用 `int`。
- **大数据下少写没必要的拷贝**：例如遍历大对象时可写 `for (auto &x : v)`，避免复制。
